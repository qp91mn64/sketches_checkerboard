# 点击鼠标画图

把画布分成网格，分别画不同图案。画图参数事先准备好，统一存放，既可以根据坐标算出，也可以来自外部文件。数值不同，图就不同，就能组合各种更加复杂的图形。

如果点一下鼠标，图案实时变化，是不是更直观一点？数据与图案是对应的，能不能单独取出来备用？

## 目前代码

一个草图实际上定义了一个图案集。各草图支持的图案类型如下：

- `checkerboard_cell_interaction`：棋盘格
- `checkerboard_cell_interaction_2`：棋盘格
- `checkerboard_cell_complementary`：棋盘格 + 黑白互补
- `c_XOR_3_I`：棋盘格图案推广
- `c_XOR_3_I_C`：棋盘格图案推广 + 黑白互补

使用方式：点鼠标左键/右键，点哪里换哪里的图案，看能拼接出什么。按 `s` 键保存数据、图形，`r` 键重置画布。

数据看起来比较抽象，如果换个草图加载，借助不同图案集可视化呢？进一步地，自己写一个画图案的参数，自定义图案集画图，如何？同时可以比较一下不同层次的美。

如果点击鼠标太麻烦为什么不试一下用 `grid.loadData()` 加载你的数据文件？

改变画布尺寸，区域行数 `w`、区域列数 `h`，可得到单个图案大小不同的图案集。

如果你把画布调得太大，部分区域可能就点不到，从而得到的图片比较空不好看；此外部分图案组合可能有点眼花缭乱。

注意**没有撤回功能**。

此外不同草图画出的棋盘格左上角默认的颜色可能不同。

## 拆分与复用代码

- `checkerboard_cell_interaction_2`：拆分出类，受到 DeepSeek 启发使用抽象类，**便于复用**。同时加上了 `checkerboard_cell_with_load_data` 演示的加载外部数据的功能以备用。
- `checkerboard_cell_complementary`：扩展到负数
- `c_XOR_3_I`：复用 `checkerboard_cell_interaction_2` 的代码结构
- `c_XOR_3_I_C`：大多数代码复用自之前的草图 `c_XOR_3_I` 和 `checkerboard_cell_complementary`，稍微改一改就出来了，顺便整理了一下

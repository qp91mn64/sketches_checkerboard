# 位运算

- `c_XOR`：棋盘格的本质是异或？按位异或画棋盘格。

## 引入索引值 `a` 多取几位，扩展图案类型

- `c_XOR_2`：按位异或
- `c_AND_2`：按位与
- `c_OR_2`：按位或

使用方式：点鼠标改变 `a` 值看不同图形，其中左键加 1 右键减 1。可以对照控制台输出值与画出的图形。按 `s` 键保存图片。

### 预料之外的分形：类似谢尔宾斯基三角形

`c_AND_2` 就能画，就像例子 `c_AND_2/examples/c_AND_2 a_511_111111111.png`。画出这种分形最快的方式，是点 2 次右键，或者直接修改 `a` 的值为 -1 再运行，不论画布多大。

目前发现了两种得到这种分形的方式：

- `c_AND_2a`：从最细节开始  
- `c_AND_2b`：从最大的空白开始  

使用方式：点几下鼠标，对照控制台输出值与画出的图形，看一看这种图形怎么得到的。按 `s` 键保存图片。

### 比较不同位运算的图形

- `c_bitwise_1`：一次只画一种位运算
- `c_bitwise_2`：放一起比较

使用方式：点鼠标改变 `a` 值看不同图形，其中左键加 1 右键减 1。可以对照控制台输出值与画出的图形。按 `s` 键保存图片。  
对 `c_bitwise_1`，按数字 `1`、`2`、`3` 键看不同图形，分别对应按位与、按位或、按位异或，默认按位异或

## 直接填充颜色，画出新图案

位运算结果直接填充颜色，于是得到各种不同灰度

- `c_bitwise_3`：画灰度图，一次只画一种位运算
- `c_bitwise_4`：画灰度图，放一起比较

使用方式：点鼠标改变 `a` 值看不同图形，其中左键加 1 右键减 1。可以对照控制台输出值与画出的图形。按 `s` 键保存图片。  
对 `c_bitwise_3`，按数字 `1`、`2`、`3` 键看不同图形，分别对应按位与、按位或、按位异或，默认按位异或

- `c_bitwise_3a`：图案动态变化，只画灰度，夹杂不同图案划过留下的痕迹
- `c_bitwise_3b`：图案动态变化，不是灰度的都画成不透明，能看到蓝色，如果有足够耐心，能看到其他颜色
- `c_bitwise_3c`：图案动态变化，能看到蓝，黄，以及逐渐过渡到其他颜色

使用方式类似与 `c_bitwise_3` 类似。

位运算结果取平方，再对 `a` 按位与，能画出相当复杂精美的图案，`a` 变化时，图案也能动态变化。

- `c_bitwise_5`：256 到 16777215 之间的颜色不透明
- `c_bitwise_5a`：跳过 256 到 16777215 之间的颜色，聚焦于灰度图案；带有各图案留下的痕迹，同时能把不同图案放一张图里面
- `c_bitwise_5a1`：灰度，浅灰背景，查看图案范围如何一点点扩大。注意切换不同图形再切换回来，之前累积的图案范围会丢失
- `c_bitwise_5b`：更大画布，彩色

**注意：`c_bitwise_5a1`的输出图片有可能会被某些图片查看工具误认为有水印，而原始代码里面没有专门添加任何水印的部分。**

如果考虑透明度呢？`x`、`y` 比较小的区域看不出与只画灰度的明显差别；`x`、`y` 大于等于 4096 开始，其位运算结果的平方能达到 16777216，对应透明度不再是 0，就开始逐渐出现各种彩色了。此外借助 `PImage`，然后直接 `save()` 保存画布上的东西，就能保存各种半透明的颜色累加之后的效果了。

- `c_bitwise_6`：包括透明度，加上偏移量，画 4096 的附近值看半透明彩色
- `c_bitwise_6a`：512x512 像素，灰度 + 半透明彩色，画 4096 的附近值
- `c_bitwise_6b`：512x512 像素，灰度 + 半透明彩色，此外 offset 从 4096 推广到 2 的整数次幂，按位左移或者按位右移能得到的数字，直到 -2147483648。

如果拖拽图形的时候不清屏，留下旧图案，是不是就能把图案当作画笔来用了？

- `c_bitwise_6c`：alpha 非零就置 255
- `c_bitwise_6c1`：alpha 非零就置 255 + 512x512 画布
- `c_bitwise_6d`：灰度 + 半透明彩色，模糊颜色效果
- `c_bitwise_6d1`：灰度 + 半透明彩色，模糊颜色效果 + 512x512 画布

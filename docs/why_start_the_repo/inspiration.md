# 灵感来源

从**自己画的图形**说起。

几天前我用画图软件从空白画布的左上角开始，一个像素一个像素地画棋盘格，每个小格子只有一个像素。

画到 16x16 像素大小之后，模仿之前写的代码小合集 [sketches_black_white](https://github.com/qp91mn64/sketches_black_white) 里面倍增的图形模式，把小格子边长翻一倍，也画棋盘格，也组成 16x16 的区域，在第一个区域的右边，下面，右下角各画一块，画完之后一共 32x32 像素。

接着继续倍增小格子边长，再画在右侧和下方，直到画到边长 8 像素，一共占据了 64x64 像素区域的时候，边长再倍增，每个 16x16 的区域就只剩一个格子了，不如就当作单个格子画棋盘格。

然后从边长 16 的格子开始，把已经画出的模式重复了一遍（到后面就是复制粘贴），最后的图形就有 1024x1024 像素，而且有两层图形：

![图片](black_white_1.png)
图 black_white_1.png

然后就是**用代码画棋盘格**了。于是有了 `checkerboard_1.pde`、`checkerboard_2.pde`，其中边界情况有两种处理方式，可以根据喜好任选一种。

实际上，`checkerboard_2.pde` 画出的图案，小格子长宽不等，不同于常见棋盘格，不过可以视为推广得到的图案，留着备用。

此外，小方格边长 `a` 决定了棋盘格的疏密程度，倍增 `a`，图案疏密程度变化很明显，边界也十分整齐，恰好可以表达不同数值。

如果再**把整个画布也分成若干区域，每个区域填入一个数字，按照数字大小画格子边长倍增的棋盘格**呢？

为了方便，定义区域坐标 (`x`，`y`)，与像素坐标类似，只是以一整个区域作为一个单位计算坐标值。

进一步地，**填入的数字也可以用二维列表存储**，而且第一个索引值恰好对应 `y`，第二个索引值恰好对应 `x`。

于是有了 `checkerboard_cell_1.pde`，每个区域按照坐标 (`x`，`y`) 填入的数字 `data[y][x]` 满足：

`data[y][x] = x + y`

画出的图形，其中的棋盘格区域排列整齐，疏密程度渐变，自带美感。

我还尝试了位运算（[sketches_black_white 里面有详细说明文档](https://github.com/qp91mn64/sketches_black_white/tree/main/docs)，此处不再赘述），发现**按位异或**，即 `data[y][x] = x ^ y`，画出的图形更复杂而精致，于是有了 `checkerboard_cell_2.pde`。

进一步地，都能预先计算数据了，能不能**加载外部数据**，或者**保存数据为文件**呢？于是有了 `checkerboard_cell_with_load_data.pde`、`checkerboard_cell_with_save_data.pde`。

接着，如果**点一下鼠标就能改变图案**就好了，于是有了 `checkerboard_cell_with_interaction.pde`。最大的问题除了画布稍微太大了一点以外就是可能会看起来有一点眼花缭乱。
